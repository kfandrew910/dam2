Funciones

Una forma de trabajar con parte de código que va a repetirse varias veces son las funciones.

La orden return, devuelve un valor en el punto en el que se ha realizado la llamada a la función.

onClick() y document.getElementByID()

Se puede añadir la etiqueta onclick a las etiquetas de html. De esta forma, cuando se haga click en alguna de ellas, se realizará una acción.

Existe otra forma de capturar los eventos onclick, escuchando los eventos. De esta forma se pueden asignar varias funciones al mismo evento. Esta suele ser la forma de capturar eventos en la web. Se deja como ejercicio buscar cuál es este procedimiento y cuáles son los eventos que se pueden escuchar.
Una vez que se obtiene el elemento se puede manipular. Por ejemplo, con el atributo innerHTML se cambia el html que contiene una etiqueta si se mezclan onclick y document.getElementById().

Funciones que devuelven valores.

Para que la función devuelva un valor se usa return.

Una función puede varios return, pero cuando se llegue a ejecutar el primero, el programa sale de la función y no continua evaluándola. Podemos utilizarlo dentro de la estructura if - else para devolver diferentes respuestas.

Visibilidad de variables.

Las variables que se definen dentro de una función se destruyen al salir de ella. Las variables definidas fuera de una función y que se van a modificar dentro de una función se denominan variables globales. Se recomienda no usarlas o usarlas lo menos que se pueda, pues son fuente de errores y complican la lectura de los programas. Lamentablemente en JavaScript hay que usarlas en muchas ocasiones.

Funciones como variables.

Una característica útil es la de poder almacenar a las funciones como si fueran variables, e incluso pasarlas como argumentos de funciones. Estas funciones no tienen nombre por lo que se las conoce como funciones anónimas. Como actúan como variables se pueden pasar como argumentos de otras funciones. Las funciones como variables se suelen usar mucho para tratar eventos o definir objetos en Javascript.

Verificar Formularios.

En muchas ocasiones, antes de enviar un formulario, hay que verificar que sus campos sean correctos. Para ello hay que especificar la función que se va a ejecutar en el momento de pulsar el botón de enviar el formulario. Dicha función debe devolver true o fase en función de si el formulario es válido o no. Esto se especifica en el campo "onsubmit" del formulario.

Añadir o quitar clases de un elemento del DOM.

Antes se ha visto que se puede cambiar el estilo de un elemento de la página, obteniendo dicho elemento con getElementById y después usando el atributo style. También se pueden añadir o quitar clases CSS de un objeto con el atributo classList.add('nombre de la clase') y classList.remove('nombre de la clase').
